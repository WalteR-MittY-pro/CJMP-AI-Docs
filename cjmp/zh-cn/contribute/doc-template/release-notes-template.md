# Release Notes 写作模板

## 总体写作说明
> **说明：**
> 所有的写作说明，在完成写作后，都要删除。

Release Notes（发布说明）应清晰、准确地描述版本变更内容，帮助开发者快速了解新版本的主要变化、升级方法和注意事项。写作时应遵循以下原则：

1. **准确性**：确保所有信息准确无误，特别是版本号、日期、commit ID等关键信息
2. **完整性**：涵盖所有重要的变更内容，不遗漏关键信息
3. **清晰性**：使用简洁明了的语言，避免歧义
4. **实用性**：提供足够的升级指导和问题规避方案
