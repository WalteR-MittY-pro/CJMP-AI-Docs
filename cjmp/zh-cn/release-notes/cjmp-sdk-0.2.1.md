### 3.3 Test
#### 3.3.1 新增特性
- 本仓为新增仓，主要功能是提供统一跨平台 UI 自动化 API（Android/iOS/OHOS 共用一套 CJ 调用方式），允许用户在业务 app 内直接跑 UI 测试。当前主要用于 Android&iOS 平台测试。

#### 3.3.2 已知问题

- **问题1：Android端，在子线程中启动测试会闪退**
    - **现象描述**：Android端，在后台线程中启动测试会闪退
    - **触发条件**：Android手机，使用spawn启动子线程，并在子线程中调用UI测试框架相关API
    - **影响范围**：仅影响Android端, iOS端正常
    - **规避方案**：暂无
    - **计划修复版本**：v0.2.2

- **问题2：driver.click点击坐标位置偏移**
    - **现象描述**：通过driver.click点击坐标位置偏移，点击的位置与期望位置不一致
    - **触发条件**：当手机顶部存在系统状态栏时，由于测试框架识别的坐标位置是基于Engine的framenode树，并不包括顶端原生的状态栏，就会导致实际界面坐标和测试框架识别的坐标位置不一致，从而导致点击偏移。
    - **影响范围**：Android & iOS端
    - **规避方案**：暂无
    - **计划修复版本**：v0.2.2

### 3.5 Tools

#### 3.5.1 新增特性

- **Logic-module 支持对外暴露原生接口**
  - **功能描述**：logic-module-native 模板工程中的 logic-module 模块现已支持将接口直接暴露给原生层，Native 壳工程无需桥接即可直接调用。
  - **使用场景**：适用于将 logic-module 集成到现有原生工程中，消除桥接转换成本，实现无缝接入，提升开发效率。

- **CJMP 工程新增 UI Test 功能**
  - **功能描述**：CJMP 工程支持在 cj 文件中通过 `import ohos.ui_test` 引入 UI 测试框架，启用组件级测试能力。
  - **使用场景**：用于组件级别的自动化测试，验证 UI 组件的行为是否符合预期。

- **Android 侧支持原生 IDE 构建流程**
  - **功能描述**：CJMP 工程可直接使用 Android Studio/Gradle 完成 Android 侧源码级构建，无需依赖命令行工具。
  - **使用场景**：适用于习惯使用 Android Studio 进行开发的场景，降低命令行与 IDE 构建差异，提升开发效率。

- **App 模板新增 `--ios-export-method` 参数**
  - **功能描述**：创建 app 工程时新增 `--ios-export-method` 参数，默认值为 `debugging`，支持个人开发者账号构建 IPA。
  - **使用场景**：个人开发者无需企业证书即可完成 iOS 应用的打包和分发。

#### 3.5.2 变更特性

- **工程类型命名调整：`package` -> `logic-module`**

  | 变更前 | 变更后 | 是否兼容变更 | 变更说明 |
  |--------|--------|--------------|----------|
  | `keels create --package` / `keels create -t package` | `keels create --logic-module` / `keels create -t logic-module` | 否 | 工程类型语义统一为 `logic-module`，更准确地表达模块用途，原参数不再支持。 |

- **Android 最低支持版本调整**

  | 变更前 | 变更后 | 是否兼容变更 | 变更说明 |
  |--------|--------|--------------|----------|
  | Android 默认 target 为 `aarch64-linux-android31`，支持 Android 12.0+ | Android 默认 target 调整为 `aarch64-linux-android26`，支持 Android 8.0+ | 否 | 与当前 SDK 支持矩阵（Android 8.0+）对齐，构建链路更加统一。 |

- **移除 combined 特性**

  | 变更前 | 变更后 | 是否兼容变更 | 变更说明 |
  |--------|--------|--------------|----------|
  | 模板支持 `combined` 特性配置 | 模板中移除 `combined` 相关配置 | 否 | 仓颉工具链在 macOS 上不支持 `combined` 特性，为避免构建问题，统一移除该配置。 |

#### 3.5.3 修复问题

- 修复了工程路径中包含空格时命令执行失败的问题。
- 修复了工程无法按需加载动态库的问题。
- 修复了 logic-module-native 工程不支持 iOS 模拟器的问题。
- 修复了 logic-module 工程 iOS 端首次运行闪退的问题。
- 修复了非 combined 场景下 iOS 编译失败的问题。
- 修复了模拟器按 ID 启动误匹配的问题。

#### 3.5.4 已知问题

- **问题1：暂不支持iOS侧链接静态库**
  - **现象描述**：CJMP工程在iOS端仅支持动态库集成方式，无法直接链接和使用静态库。
  - **触发条件**：在CJMP模块的配置或代码中引入了静态库（.a）作为依赖，并尝试在iOS端构建和运行。
  - **影响范围**：影响使用CJMP进行iOS应用开发、构建、上架的开发者。
  - **规避方案**：暂无。
  - **计划修复版本**：v2.1.3

### 3.7 DevTools

#### 3.7.1 新增特性

- 模拟器支持性能采集
    - 功能描述：支持模拟器app的性能采集能力
    - 使用场景：指定Android、IOS、OHOS某平台模拟器上某个app
    - 示例代码：
    ```
    // 启动采集（与真机采集一致）：platform参数必须指定、app和pid参数二选其一指定、device若不指定则取设备列表首个
    devtools cpu start --platform [ios/android/ohos] --app <appid> --pid <pid> --device <deviceid>
    // 停止采集并输出报告
    devtools cpu stop
    ```

## 6. 升级指导

### 6.1 升级前准备

1. 备份当前项目代码和配置文件
2. 确认当前开发环境满足新版本要求（参见SDK兼容性说明）
3. 下载并安装配套的仓颉工具链版本

### 6.2 升级步骤

```bash
# 步骤1：更新SDK版本
# 请根据实际项目配置更新SDK版本号至v0.2.1

# 步骤2：更新依赖
# 同步项目依赖，确保所有依赖项与新版本兼容

# 步骤3：检查代码兼容性
# 根据变更特性章节的迁移指导，检查并更新不兼容的代码
```

### 6.3 升级后验证

1. 编译项目，确保无编译错误
2. 运行单元测试，验证核心功能正常
3. 在目标平台（HarmonyOS/Android/iOS）上进行功能测试
